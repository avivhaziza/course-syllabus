▾
🗣 מה לומר
״כולנו מכירים Claude.ai — נכנסים לאתר, כותבים שאלה, מקבלים תשובה. זה נהדר, אבל זה רק צ'אט. Claude Code זה משהו אחר לגמרי.״
👁 מה להראות
הצבע על שלושת הטאבים באפליקציה: Chat, Cowork, Code
🗣 מה לומר
״שלושה מצבים שונים באפליקציה אחת. Chat — זה Claude הרגיל שכולם מכירים. Cowork — עוזר עם קבצים ומסמכים. Code — זה מה שאנחנו עובדים איתו היום. ב-Code, Claude לא רק עונה — הוא פועל. הוא יוצר קבצים על המחשב שלכם, הוא כותב קוד, הוא מריץ אותו, הוא מתקן שגיאות — הכל בתוך האפליקציה הזו, ללא Terminal, ללא התקנות מסובכות.״
🗣 מה לומר
״אני עובד לפי גישה שאני קורא לה: אתם האדריכל, Claude Code הוא הקבלן. אתם מחליטים מה לבנות ואיך זה אמור לעבוד — הוא בונה. לא צריך לדעת לקרוא קוד. צריך לדעת לתקשר בצורה ברורה.״

🗣 מה לומר
״לפני שניגע בפרויקט — אני רוצה ללמד אתכם משהו שישנה לכם את כל חוויית העבודה עם Claude Code. זה נקרא CLAUDE.md.״
🗣 מה לומר
״תדמיינו שאתם שוכרים קבלן מצוין לשיפוץ הבית. ביום הראשון אתם יושבים איתו ומסבירים: מה הבית, מה כבר בנוי, מה הסגנון שאתם רוצים, מה אסור לגעת בו. Claude Code לא זוכר שיחות קודמות — כל פעם שאתם פותחים שיחה חדשה, הוא מתחיל מאפס. CLAUDE.md הוא המסמך הזה שנמצא בתיקיית הפרויקט, וכל פעם שהוא מתחיל לעבוד — הוא קורא אותו ומגיע מוכן.״
🗣 מה לומר
״מה כותבים בו? מה הפרויקט — תיאור קצר. ה-stack — Node.js, Telegraf, Anthropic API. כללי עבודה — לא לשנות קבצים בלי אישור. מבנה התיקיות. משתני סביבה. ואנחנו לא נכתוב אותו ביד — ניגש ל-Claude Code ונגיד לו: 'אנחנו מתחילים פרויקט חדש, תכין לי CLAUDE.md מתאים'. הוא יכתוב, אנחנו נאשר. ככה זה עובד.״
💡 טיפ
שאל את הקהל: ״לפי מה שאמרתם, מה הייתם כותבים ב-CLAUDE.md שלכם?״

🗣 מה לומר
״לפני שנבנה — בואו נבין את התמונה הכוללת.״
👁 מה להראות
ציור/הסבר בידיים: אתם בטלגרם ← Bot Server ← Claude API ← פעולה ← תשובה בחזרה
🗣 מה לומר
״Bot Server — שרת קטן שרץ על המחשב שלכם, מאזין כל הזמן להודעות. כשמגיעה הודעה — שולחים אותה ל-Claude API, המוח של הסוכן. Claude מחליט מה לעשות: לקרוא יומן? לקבוע פגישה? התשובה חוזרת אליכם דרך הבוט בטלגרם.״
🗣 מה לומר
״לסוכן אין גישה ישירה ל-Google Calendar שלכם — וזה בכוונה. במקום זה, פתחנו לו כתובת Gmail ייעודית. כשרוצים לקבוע פגישה — הוא שולח מייל עם קובץ זימון סטנדרטי, ה-ICS, וה-Calendar שלכם מזהה אותו אוטומטית.״
💡 טיפ
״יש דרכים ישירות יותר לחבר ל-Calendar API — אבל הן דורשות הגדרות OAuth מורכבות. הגישה הזו עובדת מצוין ל-90% מהצרכים ומהירה יותר לבנות.״

📁 הקמה: תיקייה + בוט + .env
10 דק'
▾
🗣 מה לומר
״הדבר הראשון שעושים בכל פרויקט חדש — מקשרים תיקייה. Claude Code צריך לדעת איפה לעבוד.״
👁 מה להראות
פתח Explorer ← צור תיקייה חדשה בשם telegram-calendar-bot ← חזור ל-Claude Code ← לחץ על אייקון התיקייה ← בחר את התיקייה
🗣 מה לומר
״שימו לב — בתחתית המסך רואים את הנתיב של התיקייה שבחרנו. זה אומר ש-Claude Code יודע עכשיו איפה הפרויקט שלנו.״
👁 מה להראות
עבור לטלגרם — חפש @BotFather — שלח /newbot — תן שם לבוט — קבל Token
🗣 מה לומר
״הטוקן הזה הוא ה-ID הסודי של הבוט. מי שיש לו אותו יכול לשלוח הודעות כאילו הוא הבוט שלנו. לכן לא שמים אותו בקוד — שמים אותו בקובץ .env.״
⌨️ פרומפט
קיבלתי Telegram Bot Token מ-BotFather. צור קובץ .env עם מקום לכל המשתני סביבה שנצטרך: TELEGRAM_TOKEN, ANTHROPIC_API_KEY, AGENT_EMAIL, AGENT_EMAIL_PASSWORD. צור גם .env.example ללא הערכים האמיתיים וודא שה-.env נמצא ב-.gitignore.
👁 מה להראות
הכנס את ה-Token בפועל לקובץ ה-.env
💡 טיפ
״ה-.env.example זה בשביל אנשים אחרים שיורידו את הפרויקט — הם יודעים מה צריך למלא בלי לראות את הערכים האמיתיים שלך.״

⌨️ פרומפט
אנחנו מתחילים פרויקט חדש שנקרא telegram-calendar-bot. אני רוצה לבנות בוט טלגרם שפועל כעוזר אישי לניהול יומן Google Calendar. הבוט יהיה כתוב ב-Node.js עם ספריית Telegraf, ישתמש ב-Anthropic API עם מודל Claude כמוח, ויתקשר עם Google Calendar על ידי שליחת מיילים עם ICS invites דרך Gmail ייעודי. צור לי: (1) CLAUDE.md מפורט שיסביר לך את הפרויקט בכל שיחה חדשה, (2) מבנה תיקיות מסודר לפרויקט.
👁 מה להראות
תן ל-Claude Code לעבוד — הראה לקהל שהוא יוצר קבצים ממש בתיקייה. כשמסיים — פתח את CLAUDE.md ב-Explorer וקרא בקול קטע ממנו.
🗣 מה לומר
״שימו לב למה שהוא כתב — הוא לא רק תיאר את הפרויקט, הוא גם הגדיר כללי עבודה לעצמו. זה הכלי הכי חזק שיש לכם.״

📅 הוספת ניהול יומן
15 דק'
▾
👁 מה להראות
עבור ל-Gmail החדש של הסוכן — Google Account ← Security ← 2-Step Verification — הפעל. חזור ל-Security ← App Passwords ← צור סיסמה ← העתק ← שמור ב-.env תחת AGENT_EMAIL_PASSWORD
🗣 מה לומר
״למה אנחנו צריכים App Password ולא סיסמה רגילה? כי Google לא מאפשרת לאפליקציות חיצוניות להתחבר עם הסיסמה הרגילה מטעמי אבטחה. App Password זה סיסמה מיוחדת שנוצרת רק לשימוש הזה.״
⌨️ פרומפט
הוסף ל-index.js יכולת לנהל יומן. כשהמשתמש מבקש לקבוע פגישה — הסוכן צריך לשלוח מייל מ-AGENT_EMAIL אל [המייל שלי] עם ICS invite. השתמש ב-nodemailer. ה-ICS צריך לכלול: כותרת הפגישה, תאריך ושעה, משך של שעה כברירת מחדל, וקישור Google Meet. הוסף גם יכולת לענות על 'מה יש לי ביומן' עם תשובה שמסבירה שהוא יכול לראות פגישות שהוא קבע.
👁 מה להראות
עבור לטלגרם — שלח: ״תקבע לי פגישה מחר בעשר בבוקר עם דני, נושא שיווק״. הראה שהמייל הגיע ל-Gmail שלך. הראה שהאירוע נכנס ל-Calendar. שלח: ״מה יש לי ביומן השבוע?״ — הראה את התשובה.

🗣 מה לומר

🗣 מה לומר
